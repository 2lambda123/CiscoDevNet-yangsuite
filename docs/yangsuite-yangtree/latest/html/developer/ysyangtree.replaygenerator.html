Generate an appropriate name for a constructed replay.

Note that yangsuite-testmanager identifies generated replays based on the naming pattern, so any changes to this need to be validated against that!

Parameters:
  • counter (int) – Numerical ID associated with this replay
  • prefix (str) – XPath this replay applies to
  • suffix (str) – Descriptive string for this replay
Returns:

str – Of no more than about 72 characters in length, if possible.

Examples:

# Simple case - no trimming required 1, “/native/banner/exec/banner”, “basic merge” # Returns ‘00001 - /native/banner/exec/banner - basic merge’

# XPath needs namespace prefixes stripped and to be trimmed as well 150, ‘<long path>/pfx:redundancy/pfx:minimum-preferred-links’, ‘extra’ # Returns ‘00150 - …/redundancy/minimum-preferred-links - extra’

# Also requires stripping out of list keys 1, ‘/interface[active=”act”]/physical/slot’, “basic replace” # Returns ‘00001 - …/physical/slot - basic replace’

# Both prefix and suffix need to be trimmed for length

ysyangtree.replaygenerator.extra_node_data(stmt=None, parent_data=None, node=None)[source]¶

Add tailf annotations to the node data.

Callback function for ysyangtree.ymodels.ParseYang.generate_tree().

Currently uses the following annotations:

  • tailf:cli-sequence-commands - specifies that the arguments in the model need to each be input in order to be accepted by the IOS XE CLI
  • tailf:cli-break-sequence-commands - negates a previous cli-sequence-commands annotation, meaning that this node and those after it no longer need to be specified in full order.
  • tailf:cli-incomplete-command - specifies that this leaf by itself does not constitute a complete CLI command. When combined with cli-sequence-commands, this means that the next leaf in the current container also needs to be included for the command to be considered valid.
  • tailf:cli-optional-in-sequence - specifies that the leaf may be omitted from the sequence of leafs under a cli-sequence-commands container. If not specified, this leaf must be specified if the next leaf in the container is also to be specified.

Between the above annotations, we can determine any leafs that must precede a given leaf (due to cli-sequence-commands plus the absence of any cli-optional-in-sequence annotations, analogous to a YANG “when” clause) and any leafs that must follow a given leaf (due to the given leaf being cli-incomplete-command, analogous to a YANG “must” clause).

For more information on these and other annotations, see: http://wwwin-people.cisco.com/phpetty/nso_man/man.5.tailf_yang_cli_extensions.html

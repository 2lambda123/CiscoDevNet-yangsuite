ysfilemanager.views.repository.repository_required(decoratee)[source]¶

Decorator for views that expect a ‘repository’ arg on the given request.

If the arg is provided and valid, it’s split with split_user_set() and passed into the decorated function as args ‘owner’ and ‘reponame’.

Returns:JsonResponse – if repository parameter is missing or incorrect.
